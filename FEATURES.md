# EdgeTunnel 增强功能说明文档

## 概述

本项目基于 [cmliu/edgetunnel](https://github.com/cmliu/edgetunnel) 进行功能增强，主要增加了用户管理和时间限制功能，以满足班级同学使用场景下的权限管理需求。

## 新增功能

### 1. 用户管理系统

#### 功能特点
- **用户验证**：每个用户都有独立的有效期管理
- **时间限制**：可设置用户使用天数，到期自动失效
- **管理界面**：提供可视化用户管理面板
- **API接口**：支持用户创建、更新、删除、查询等操作

#### 核心功能函数
- `validateUserExpiration(env, userId)` - 验证用户是否过期
- `createUser(env, userId, days)` - 创建新用户并设置使用期限
- `updateUser(env, userId, days)` - 更新用户使用期限（续期）
- `deleteUser(env, userId)` - 删除用户
- `getAllUsers(env)` - 获取所有用户列表

### 2. 时间限制机制

#### 实现原理
- 使用 Cloudflare Workers KV 存储用户信息
- 每个用户分配独立的过期时间戳
- 订阅请求时验证用户有效性
- 过期用户无法访问服务

#### 默认设置
- 默认用户有效期：30天 (`USER_EXPIRATION_DAYS_DEFAULT`)
- 过期后自动禁用订阅访问
- 提供友好的过期提示信息

### 3. 管理API接口

#### 用户管理API
- `POST /admin/users/create` - 创建用户
- `POST /admin/users/update` - 更新用户（续期）
- `POST /admin/users/delete` - 删除用户
- `POST /admin/users/list` - 获取用户列表

#### 请求格式
```json
{
  "userId": "用户ID（可选，默认使用当前用户ID）",
  "days": "使用天数（必需）"
}
```

### 4. 订阅访问控制

#### 过期检查
- 在订阅请求处理前进行用户有效性检查
- 过期用户返回403错误状态码
- 显示"用户已过期，请联系管理员续期"提示

## 部署说明

### 环境变量配置
- `USERS_KV` - 必需：Cloudflare Workers KV命名空间，用于存储用户数据
- `ADMIN` 或 `PASSWORD` - 管理员密码，用于访问管理面板
- `KEY` - 加密密钥，用于安全验证

### 部署步骤
1. 在 Cloudflare Workers 页面创建新的KV命名空间，名称为`USERS_KV`
2. 在项目设置中绑定KV命名空间到环境变量`USERS_KV`
3. 设置管理员密码等必要环境变量
4. 部署项目

## 使用说明

### 管理员操作
1. 访问 `/admin` 进入管理面板
2. 使用管理员密码登录
3. 在管理面板中进行用户管理操作

### 用户操作
- 用户访问订阅链接时会自动验证有效期
- 过期用户无法获取订阅内容

## 安全特性

### 权限控制
- 只有管理员可以访问用户管理功能
- 用户只能访问自己的订阅内容
- 过期用户自动失去访问权限

### 数据保护
- 用户数据存储在安全的KV存储中
- 订阅访问受到严格验证
- 过期机制防止无限期使用

## 技术架构

### 数据存储
- 使用 Cloudflare Workers KV 存储用户信息
- 用户数据包括：ID、创建时间、过期时间、总天数

### 验证流程
1. 用户发起订阅请求
2. 系统提取用户ID
3. 查询KV获取用户信息
4. 验证用户是否过期
5. 根据验证结果决定是否提供订阅内容

## 维护说明

### 日常维护
- 定期检查用户列表
- 根据需要为用户提供续期服务
- 监控系统运行状态

### 故障排查
- 如果用户无法访问，检查用户是否过期
- 如果管理面板无法访问，检查管理员密码设置
- 如果功能异常，检查KV绑定是否正确

## 注意事项

1. 此功能专为教育用途设计，仅限班级同学使用
2. 请定期备份用户数据（通过KV管理界面）
3. 合理设置用户使用期限，避免频繁续期
4. 遵守当地网络使用政策和规定
